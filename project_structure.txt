
# هيكل مشروع بوت خدمات السوشيال ميديا
==============================================

## المجلدات الرئيسية:

### 1. database/ - قاعدة البيانات
   ├── __init__.py      - ملف تهيئة المجلد
   ├── core.py          - وظائف أساسية لقاعدة البيانات (إنشاء/تحديث/حذف)
   ├── deposit.py       - إدارة عمليات الإيداع
   └── ranks.py         - إدارة رتب المستخدمين

### 2. handlers/ - معالجات الأحداث
   ├── __init__.py      - ملف تهيئة المجلد
   ├── admin.py         - معالجات الأوامر الخاصة بالمشرفين
   └── user.py          - معالجات الأوامر الخاصة بالمستخدمين العاديين

### 3. keyboards/ - لوحات المفاتيح
   ├── __init__.py      - ملف تهيئة المجلد
   ├── inline.py        - لوحات المفاتيح المضمنة (Inline Keyboards)
   └── reply.py         - لوحات المفاتيح العادية (Reply Keyboards)

### 4. services/ - خدمات خارجية
   └── api.py           - التفاعل مع واجهة برمجة التطبيقات الخارجية

### 5. states/ - حالات المحادثة
   └── order.py         - حالات الطلبات والإيداع

### 6. utils/ - أدوات مساعدة
   ├── __init__.py      - ملف تهيئة المجلد
   ├── common.py        - وظائف مشتركة ومساعدة
   └── keyboard_manager.py - إدارة لوحات المفاتيح

### 7. logs/ - مجلد لملفات السجل

## الملفات الرئيسية:

1. main.py              - نقطة بدء تشغيل البوت
2. bot.py               - تهيئة البوت وتسجيل المعالجات
3. config.py            - ملف الإعدادات والتكوين
4. requirements.txt     - قائمة المكتبات المطلوبة
5. .replit              - ملف تكوين بيئة Replit
6. replit.nix           - ملف تكوين حزم Nix لـ Replit

## وصف المكونات الرئيسية:

### قاعدة البيانات (database/):
- إدارة بيانات المستخدمين
- تتبع الطلبات والمعاملات
- إدارة عمليات الإيداع والسحب
- إدارة رتب المستخدمين وميزاتها

### معالجات الأحداث (handlers/):
- معالجة أوامر المستخدمين
- معالجة أوامر المشرفين
- التعامل مع واجهة المستخدم

### لوحات المفاتيح (keyboards/):
- إنشاء وإدارة لوحات المفاتيح التفاعلية
- لوحات مفاتيح للمستخدمين العاديين
- لوحات مفاتيح خاصة بالمشرفين

### الخدمات (services/):
- التفاعل مع API لخدمات السوشيال ميديا
- إرسال واستقبال البيانات من الخدمات الخارجية

### حالات المحادثة (states/):
- إدارة سير العمل للطلبات
- إدارة عمليات الإيداع
- التحكم في تدفق المحادثة

### الأدوات المساعدة (utils/):
- وظائف مساعدة عامة
- تنسيق النصوص والأرقام
- إدارة سجلات الأحداث
- معالجة استثناءات

## التدفق الرئيسي للبوت:
1. يبدأ المستخدم محادثة مع البوت (start)
2. يمكن للمستخدم:
   - الاطلاع على الخدمات المتاحة
   - إنشاء طلب جديد
   - إيداع رصيد
   - عرض طلباته السابقة
   - التواصل مع الدعم
3. المشرفون لديهم واجهة إضافية لإدارة:
   - المستخدمين
   - الطلبات
   - عمليات الإيداع
   - إحصائيات النظام
